# 健身跑步知识库构建计划

## 1. 知识点结构设计 (Knowledge Taxonomy)
为了确保分类明确且无死角，我们将知识库划分为以下核心模块：

*   **运动生理学基础 (Exercise Physiology)**
    *   能量代谢系统 (ATP-CP, 糖酵解, 有氧氧化)
    *   肌肉纤维类型与适应机制
    *   心肺功能与最大摄氧量 (VO2 Max)
*   **跑步科学 (Running Science)**
    *   跑姿生物力学 (Biomechanics)
    *   训练周期化理论 (Periodization)
    *   伤病预防与康复 (Injury Prevention)
    *   装备科技与选择 (Footwear & Gear)
*   **力量与体能训练 (Strength & Conditioning)**
    *    hypertrophy (肌肥大) 机制
    *   神经肌肉适应与力量增长
    *   功能性训练与核心稳定性
*   **营养与补剂 (Nutrition & Supplementation)**
    *   宏量营养素配比与运动表现
    *   补剂循证分析 (Creatine, Caffeine, Beta-Alanine等)
    *   水合状态与电解质平衡
*   **恢复与心理 (Recovery & Psychology)**
    *   睡眠对运动表现的影响
    *   过度训练综合征 (OTS) 监测
    *   运动心理学与动机维持

## 2. 自动化搜寻计划 (Automated Search Strategy)
利用智能体执行以下搜寻任务，确保资料的真实性与前沿性：

*   **学术文献挖掘**: 
    *   针对每个子分类，设定关键词在 PubMed 和 Google Scholar 进行检索。
    *   筛选标准：近 5 年发表、引用率高、随机对照试验 (RCT) 或 Meta 分析优先。
*   **专业平台整合**: 
    *   从 Examine.com, Stronger By Science, Running Research News 等平台提取经过同行评审的科普内容。
*   **数据清洗与结构化**: 
    *   自动提取论文摘要、结论及关键数据。
    *   生成 Markdown 格式的报告，包含“核心结论”、“实验设计”、“实际应用建议”三个板块。

## 3. 网站架构与排版设计 (Website Architecture)
基于 React + Vite 构建，强调学术性与舒适度：

*   **技术选型**: React (UI), Vite (Build), Tailwind CSS (Styling), React Router (Navigation).
*   **页面布局**: 
    *   **首页**: 模块化导航卡片，展示最新收录的研究报告。
    *   **知识库详情页**: 左侧为目录树，右侧为正文。支持 LaTeX 公式渲染（用于生理指标）和 Mermaid 图表（用于流程图）。
    *   **参考文献索引**: 每篇文章底部自动生成 APA 格式的引用列表。
*   **视觉风格**: 采用极简主义设计，使用衬线字体增强学术感，配色以低饱和度的蓝灰色为主，减少视觉疲劳。

## 4. 实施步骤
1.  **初始化项目**: 搭建 React + Vite 基础框架。
2.  **执行搜寻智能体**: 运行自动化脚本，按分类抓取并整理文献报告至 `data/reports` 文件夹。
3.  **内容解析引擎**: 开发 Markdown 解析组件，将搜集的报告动态渲染到前端。
4.  **UI/UX 优化**: 完善分类导航、搜索功能及响应式排版。
